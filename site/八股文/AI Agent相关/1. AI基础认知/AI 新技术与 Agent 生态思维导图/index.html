AI 新技术与 Agent 生态知识图谱

模块化拆解 · 关系型认知 · 核心概念速览

架构化认知提示： 这十二个名词并非孤立存在。 “协议”（MCP/A2A）是底层的连接标准； 基于这些标准和封装好的“能力组织方式”（Skills/Workflow）， 诞生了多样化的“Agent 产品”（Claude Code/Manus 等）； 而这些工具和生态反过来塑造了人类在 AI 时代的“AI 编程与工作方式”（Vibe Coding）。

01. 协议 (Protocol)

连接的基石，解决“如何打通”与“如何协同”的问题

底座标准

MCP (Model Context Protocol)

大模型连接外部工具和数据的标准协议。你可以把它理解为 AI 世界的“通用插座”。

核心解决：以往每个模型、每个软件都要写一套私有的接口去连接外部文件、数据库或内部系统。通过 MCP，各大模型可实现“即插即用”式连接，消除定制开发的繁琐，极大地丰富了模型获取上下文的能力。

A2A (Agent-to-Agent)

Agent 与 Agent 之间通信与协作的协议。如果说 MCP 解决的是“怎么连接工具”，那 A2A 解决的就是“Agent 之间如何沟通和分工”。

核心解决：在多 Agent 系统（Multi-Agent）中，需要一套公共标准的语意和协作格式，使擅长搜索的 Agent 与擅长报表、写代码的 Agent 能无缝咬合、高效流转，从而交付超出单体 Agent 能力极限的复杂成果。

02. Agent 产品 (Products)

面向特定场景或通用场景的自主执行体

落地形态

Claude Code

Anthropic 推出的终端本地 Coding Agent。它不仅是代码补全，更偏向于“自主帮你完成开发任务”。

核心特点：它运行在终端命令行，能深度理解整个代码库（Repository），可自主跨多个文件修改、跑本地命令、运行测试、解决报错、管理 Git 提交。开发者的角色从“逐行编码”转变为“提需求和审核结果”。

Codex

OpenAI 的云端软件工程 Agent。与 Claude Code 本地路径不同，它更强调“云端沙箱隔离与多任务交付”。

核心特点：在云端沙箱环境（Sandbox）中拉取代码仓库，并发运行各种自动化任务（如漏洞修复、新功能开发、持续集成测试），最终直接向代码库交付干净、可合并的 Pull Request。

Manus

结果交付型通用 Agent。它的核心特色在于直接代替人类进行多步骤的浏览器和端侧交互，“替你做事并直接产出物理成品”。

核心特点：不需要人类一步步教，只要给出大目标，它就能自己在沙箱中打开网页搜索、搜集信息、用工具编写代码，最终自主产生一份完整的 PPT、报告、可运行的网页或小应用。

AutoGLM

智谱自主执行 Agent 能力及产品体系。它是典型的“Phone Use / Browser Use”的代表产品。

核心特点：具备像人一样“看懂”手机屏幕和浏览器网页并进行模拟点击、滑动、输入的能力。在应用层，它可以帮你实现自动点外卖、自动发微信朋友圈、自动搜集整理特定行业数据。

豆包手机

AI 原生手机硬件与系统助手形态。它不是单纯的 App，而是把 AI 深度重组至“系统级交互与硬件生态”中。

核心特点：实现物理按键直接唤起、全时无缝语音唤醒、深度调用系统底层 API 和各项手机功能，提供跨 App 数据流转能力，让整个手机硬件成为承载 AI 个人助手的贴身载体。

Cowork

Anthropic 研发的面向非研发办公场景的桌面 Agent。如果说 Claude Code 专攻研发，那 Cowork 就是“白领知识工作者的虚拟助理”。

核心特点：深度整合于日常办公软件和桌面生态。专注于复杂的案头工作、海量资料多维交叉比对、日常公文撰写、跨软件审批和任务执行，旨在替非研发工作者解决繁琐流程。

OpenClaw / Clawdbot

聊天入口驱动的个人 Agent。它倡导用户不改变习惯，在日常最频繁的通讯软件中“以聊天为入口完成实际任务执行”。

核心特点：直接嵌入 WhatsApp、Telegram、Slack、飞书等成熟的社交和办公协作平台。在对话流中，它不仅扮演“回答者”，还能作为个人秘书，替你去查邮件、订日程、执行数据操作，主打“高频无摩擦”的入口体验。

03. AI 编程方式 (Coding Paradigm)

人与 AI 协作生成代码的范式转移

范式革命

Vibe Coding (氛围感编程 / 感觉流编程)

由 Andrej Karpathy 提出的 AI 时代全新软件开发范式。核心是“人出感觉和方向，AI 交付最终代码”。

核心特点：人类彻底脱离手动敲击代码、研究繁琐 API、调试细枝末节的传统状态，只需用自然语言描述自己想要的效果、界面的“氛围感（Vibe）”和功能框架，其余 99% 的底层细节、代码重构、测试用例全部由后台的 AI 自动包办。

04. Agent能力组织方式 (Architecture)

AI 内部能力的沉淀与工程化的链路编排

工程方法

Skills (技能包)

给 Agent 封装好的、高内聚且可复用的能力包。本质上是在“固化和沉淀专家经验”。

核心特点：技能包中包含了特定任务所需的 Prompt 指导、执行模版、自动化脚本和必要资源。它让 Agent 在面对类似任务时能直接激活、调用特定技能包，而不是每次都极其不稳定地重新探索提示词，极大提高了 Agent 生产环境的可靠性。

Workflow (工作流)

预先确定、刚性编排的确定性执行链路。它是与“高灵活性、动态决定下一步”的完全体 Agent 相对的一种“确定性工程链路”。

核心特点：典型的结构类似：“输入 -> 分类判断 -> 路由选择 -> 深度检索 -> 规范生成 -> 校验拦截 -> 输出”。尽管它内部的节点可以用大模型，但整体的运作脉络是不可擅自改变的，非常适用于金融、医疗、法律等对确定性、安全防线有极高要求的严肃业务场景。

生态联动与面试高频辨析

连接机制：MCP vs A2A

MCP 是 AI 与外部硬工具（文件、数据库等）之间的物理插头。
A2A 是 AI 之间用于高级商务、跨领域协作的信息握手。一个是“人机/机具接口”，一个是“社会化接口”。

产品落点：Claude Code vs Codex

两者皆属于 Coding Agent。
Claude Code 深度贴合开发者的命令行本地工作习惯，直接影响“单兵作战”的极致效率；
Codex 重在云端多任务并行的沙箱工程交付，面向企业端云原生研发。

工程实现：Skills vs Workflow

Skills 是一个封装好的“技能模块”，由 Agent 在运行时根据目标自主调配和决定要不要用。
Workflow 则是规定好方向的高速公路，AI 只能在这条确定的车道内按步骤疾驰。
